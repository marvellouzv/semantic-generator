# ARENT.md

## Назначение

Этот файл фиксирует правила работы с проектом `semantic-generator` для агента и разработчиков.

## Обязательные файлы документации

- `README.md` — продуктовая и техническая справка: что делает проект, как запустить, как проверять.
- `handoff.md` — журнал изменений для контроля и передачи контекста между сессиями.
- `arent.md` — текущий регламент работы и поддержания актуальности документации.

## Базовые правила работы

1. Перед началом изменений проверять текущие `README.md`, `arent.md`, `handoff.md`.
2. Любое заметное изменение в коде, конфигурации, запуске, API или процессе фиксировать в `handoff.md`.
3. Изменения пользовательского поведения, возможностей проекта или шагов запуска синхронно отражать в `README.md`.
4. Изменения правил командной работы отражать в `arent.md`.
5. Не хранить секреты (API-ключи, токены, пароли) в документации и в git.

## Правило по LLM-интеграции

1. Основной провайдер проекта: `ru-openrouter`.
2. В конфигурации и коде использовать в первую очередь `OPENAI_*` переменные:
   - `OPENAI_API_KEY`
   - `OPENAI_BASE_URL`
   - `OPENAI_MODEL`
   - `OPENAI_FALLBACK_MODEL`
3. Использовать OpenAI-совместимый формат API (`responses`/`chat.completions`) через единый клиент в backend.
4. При смене модели или base URL обязательно обновлять:
   - `backend/env_template.txt`
   - `README.md`
   - `handoff.md`

## Минимальный формат записи в handoff

- Дата
- Что сделано
- Затронутые файлы
- Риски/ограничения
- Что проверить после изменений

## Definition of Done для задачи

Задача считается завершенной, когда:

1. Изменения в коде внесены.
2. Проверка выполнена (локально или явно указано, что не проверено).
3. `handoff.md` обновлен.
4. `README.md` и/или `arent.md` обновлены при необходимости.
