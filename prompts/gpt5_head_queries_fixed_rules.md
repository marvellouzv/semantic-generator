CONSTRAINTS & DEFINITIONS
- «Верхнеуровневые» = короткие, наиболее частотные, обобщающие формулировки (обычно 1–3 слова, иногда 4).
- Исключать длинные хвосты, уточнения и геомодификаторы с названиями городов (Москва/СПб/Новосибирск и т.д.).
- Для локальных запросов использовать только общие формулировки БЕЗ указания конкретных городов.
- Исключать бренды, если не разрешено явно (см. блок типов).
- Нормализовать морфологию и синонимы; в Head Query оставлять каноническую форму.
- Дедупликация: объединять дублеты (ед./мн., порядок слов, опечатки, транслит), оставлять канон.
- Intent = {commercial | informational | unknown}. Это НЕ “тип запроса” из UI; это общая классификация намерения.
- Group = один из выбранных типов запросов из UI (см. блок типов), на русском. Группа может быть только из отмеченных галочкой.
- Рынок: Россия/СНГ. Язык: русский.

SCOPE & COVERAGE
- Выдать максимально возможное количество head-запросов для полного покрытия тематики (без искусственных лимитов).
- Организовать в такое количество Parent Theme, сколько нужно для полного покрытия; внутри — сортировка по ожидаемой частоте/ценности.
- Не расширять искусственно редкими формулировками: фокус на «ядре» спроса.

COVERAGE QUOTAS
- Определи столько Parent Theme, сколько нужно, чтобы закрыть всю нишу полностью.
- В каждой Parent Theme выдай столько head-запросов, сколько нужно для полного покрытия.
- Первые 3–4 head-запроса внутри каждой Parent Theme должны начинаться с разных корневых слов (не повторяй один и тот же лемматизированный корень).
- Обязательно покрой все основные аспекты тематики: услуги, цены, качество, проблемы, локацию, бренды (если разрешены).

DIVERSITY GUARDS
- Не повторяй одно и то же словоосочетание в разных Parent Theme, если это не отраслевой генерик.
- Избегай цепочек из одних и тех же производных («установк...» × 3 подряд и т.п.) — разбавляй соседними кластерами.
- Если формулировки синонимичны, оставь наиболее общеупотребимую, другие — не выводи.
- Если head уже появился в другой Parent Theme — не повторяй, замени на ближайший синонимичный, но более общий.

QUALITY RULES
- Ясность: лаконичные, общеупотребимые формулировки без хвостов.
- Полнота: покрыть все очевидные ветви спроса.
- Чистота: удалить нерелевантные/бытовые запросы.
- Финальный контроль: убрать дубликаты, добавить пропущенные очевидные вершины.

OUTPUT (STRICT)
1) Сначала одним списком выведи Parent Theme (только названия категорий), через запятую.
2) Затем ОДНА Markdown-таблица со столбцами ровно:
   - Head Query
   - Intent
   - Group
   - Demand Level
   - Parent Theme
   Без дополнительных столбцов/примечаний/пояснений.
3) Не выводи plain-text список и пост-скриптумы.

STEP-BY-STEP
1) Если тема однозначна — сразу работай (без вопросов).
2) Определи столько Parent Theme, сколько нужно (закрыть все ключевые ветви спроса).
3) Для каждой Parent Theme сгенерируй столько head-запросов, сколько нужно (1–3 слова; в исключениях до 4).
4) Нормализуй/дедуплицируй, оставь канон.
5) Проставь Intent (commercial/informational/unknown), Group (строго из разрешённых) и Demand Level (High/Medium/Low) по здравому смыслу.
6) Отсортируй внутри каждой Parent Theme по приоритету.
7) Выведи результат строго по OUTPUT (STRICT).
