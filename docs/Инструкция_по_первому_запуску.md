## Пошаговое руководство для студентов: как запустить «Semantic Generator» локально

Этот гайд рассчитан на людей **без опыта программирования**. Делайте шаги **по порядку**.

---

### 1. Что вы получите в результате

Вы запустите локальный сервис, который:

- Генерирует **SEO‑кластеры** (Этап 1: “Настройка” → “Кластеры”).
- Помогает **фильтровать, редактировать и экспортировать** результаты.
- Умеет работать с **шаблонами кластеров** (сохранить/загрузить).
- Имеет **историю генераций** (экспорт/импорт и сравнение).
- Опционально: кэширование в Redis (если Redis установлен).

---

### 2. Что нужно заранее (минимум)

- **Cursor 2.0** (или другой редактор, но в курсе мы используем Cursor).
- **Python 3.11+**
- **Node.js 18+**
- **OpenAI API Key** (обязательно, если вы хотите генерацию через GPT‑5)

Важно: установка Cursor/Python/Node.js и получение OpenAI API‑ключа уже разобраны в **отдельных уроках на платформе GetCourse**.

---

### 3. Какие API нужны и где их брать

#### 3.1. OpenAI API (обязательно для генерации)

- **Что нужно**: API‑ключ OpenAI (переменная `OPENAI_API_KEY`)
- **Где взять**:
  - Ключи: [OpenAI API keys](https://platform.openai.com/api-keys)
  - Биллинг/лимиты: [OpenAI Billing](https://platform.openai.com/settings/organization/billing)

Важно:
- Ключ — это **как пароль**. Его нельзя показывать в видео, пересылать в общий чат или коммитить в Git.
- В этом проекте генерация работает на модели **`gpt-5`**, поэтому аккаунт должен иметь доступ к GPT‑5.
 - Важно про зависимости: backend использует **Responses API** (`AsyncOpenAI.responses.create()`), поэтому нужен OpenAI SDK, где есть `responses` (см. `backend/requirements.txt`).

---

### 4. Подготовка репозитория

1. Скачайте и распакуйте репозиторий в папку без кириллицы, например:
   - `C:\seo-ai\semantic-generator-starter` (пример)
2. Откройте папку проекта в Cursor:
   - **File → Open Folder**

---

### 5. Настройка `.env` (один раз)

#### 5.1. Backend

1. Откройте папку `backend/`.
2. Скопируйте файл `backend/env_template.txt` в `backend/.env`
3. Откройте `backend/.env` и вставьте ваш ключ:

```env
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-5
```

Если у вас нет Redis — ничего страшного: сервис будет работать без кэша.

#### 5.2. Frontend

Обычно ничего менять не надо. Но если нужно — файл `frontend/.env` может содержать:

```env
VITE_API_URL=http://localhost:8000
```

---

### 6. Самый простой запуск (рекомендуется)

#### Вариант А (самый простой): автозапуск через Cursor

1) Откройте файл `docs/Промпт_для_запуска_Semantic_Generator.md`  
2) Скопируйте текст целиком → вставьте в чат Cursor → отправьте  
3) Cursor сам:
- найдёт корень проекта (без путаницы с `cd backend`/`cd frontend`);
- поставит зависимости;
- создаст `.env` из шаблонов;
- попросит вас **вставить ключ** в `backend/.env`;
- запустит backend и frontend и проверит, что всё открывается.

#### Вариант B (без автопромпта): запуск через npm-скрипты

В корне проекта:

```powershell
npm install
npm run setup
npm run dev
```

После запуска откройте:
- Backend Swagger: `http://localhost:8000/docs`
- Frontend (приложение): `http://localhost:5173`

```powershell
npm --prefix frontend run dev -- --port 5173
```

Проверка: `http://localhost:5173`

### 8. Как пользоваться интерфейсом (простое объяснение “что где”)

#### 8.1. Верхняя панель (вверху)

- **Этап 1: Настройка**: ввод темы и выбор типов запросов (интентов).
- **Этап 2: Кластеры**: таблица кластеров, фильтры, аналитика, экспорт.
- **Шаблоны**: управление шаблонами (сохранённые наборы кластеров).
- **История**: список прошлых генераций, экспорт/импорт и сравнение.

#### 8.2. Этап 1: Настройка

- **Тематика**: можно указать **несколько тем через запятую**.
- **Типы запросов (Group)**: отметьте нужные группы (коммерческие/инфо/услуги/цены/локальные и т.п.)
- **Минус-слова**: слова/фразы, которые нельзя использовать при генерации.
- **Регионы (для локальных)**: если выбраны “Локальные” и указаны регионы — локальные запросы будут только для выбранных регионов (с вариациями).
- **Массовая загрузка**: если хотите вставить большой список тем/запросов.
- **Сгенерировать кластеры**: запускает генерацию в GPT‑5.

#### 8.3. Этап 2: Кластеры

На экране “Кластеры” вы можете:
- включить/выключить **аналитику**;
- использовать **фильтры** (по интенту GPT, спросу, родительской теме, группам, тегам);
- **редактировать** название кластера;
- удалять ненужные;
- внизу экрана — **липкая панель действий** (всегда видна при прокрутке):
  - **Экспорт** (Excel/CSV)
  - **Сохранить как шаблон**
  - **Расширить выбранные**
- “Расширение запросов” находится **в самом низу страницы, под таблицей**: задать доп. требования → сгенерировать дополнительные запросы → при необходимости **добавить в таблицу**.

Важно: **Intent ≠ Group**
- **Интент (GPT)**: только `commercial | informational | unknown`
- **Группа (выбранный тип)**: только то, что отмечено галочкой на этапе настройки

### 9. Как просить Cursor исправлять ошибки и добавлять функции

Правило простое: вы описываете **что видите** и **что хотите получить**.

Можно и нужно:
- скриншотить **ошибки в терминале**, **ошибки в консоли браузера (F12 → Console)**, **кривую вёрстку**;
- прикладывать скриншоты в чат Cursor и писать: “исправь так-то”.

Готовые примеры запросов — в `10_промптов_для_студентов.md`.

### 10. Частые проблемы (самые частые)

- **Порт занят (8000 или 5173)**:

```powershell
netstat -ano | findstr ":8000 :5173"
```

Если нужно завершить процесс:

```powershell
taskkill /PID <PID> /F
```

- **Ошибка OpenAI (401/403/модель недоступна)**:
  - проверьте `backend/.env` → `OPENAI_API_KEY`
  - убедитесь, что есть доступ к модели **`gpt-5`**
  - проверьте биллинг/лимиты в OpenAI

- **Ошибка вида `AsyncOpenAI.responses not found`**:
  - значит установлен слишком старый пакет `openai`
  - решение: в активном окружении backend выполнить `pip install -U -r backend/requirements.txt`

### 11. Что делать дальше

1. Сгенерируйте кластеры под свою тему.
2. Почистите таблицу (удалить лишнее, переименовать, фильтры).
3. Сохраните как шаблон.
4. Экспортируйте результат.
